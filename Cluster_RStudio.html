1 Introducci√≥n

1.0.1 ¬øQu√© es?

El an√°lisis cl√∫ster es un m√©todo estad√≠stico multivariante de clasificaci√≥n autom√°tica no supervisada a partir de una matriz de Observaciones-variables, en grupos homog√©neos, conglomerados o cl√∫steres, no conocidos de antemano pero sugeridos, en funci√≥n de las distancias entre los puntos de datos, para obtener una mayor comprensi√≥n de los datos √≥ porcesarlos de manera m√°s eficiente. (Fuente: Shimodaira, H. (2020). Clustering and Visualisation of Data. Ed.ac.uk)

Hay dos enfoques principales para el agrupamiento: jer√°rquico y particional √≥ no Jer√°rquicos. El agrupamiento jer√°rquico forma un √°rbol de cl√∫steres anidados en el que, en cada nivel del √°rbol, un cl√∫ster es la uni√≥n de sus hijos. El agrupamiento particional √≥ no Jer√°rquico no tiene una estructura anidada o jer√°rquica, simplemente divide el conjunto de datos en un n√∫mero fijo de grupos que no se superponen, con cada punto de datos asignado a exactamente un grupo. El algoritmo de agrupamiento particional m√°s empleado, el agrupamiento K-means.

1.0.2 Objetivos del an√°lisis

  • Modo Q. Agrupar elementos en grupos homog√©neos en funci√≥n de las similitudes o similaridades entre ellos.
  • Modo R. Dividir las variables en grupos para plantear modelos formales y reducir la dimensi√≥n.

1.0.3 Consideraciones para aplicar el an√°lisis a una matriz de datos b√°sica.

Para poder aplicar el an√°lisis de importante:

  1. Verificar si el conjunto de datos no se encuentra correlacionado.
  2. Escoger de una medida de distancia propiada.
  3. Verificar la tendencia de agrupamiento del conjunto.

4.3.3 Verificar la tendencia de agrupamiento del conjunto.

La evaluaci√≥n de la tendencia de agrupacmiento se puede hacer con m√©todos num√©ricos usando el Estad√≠stico de Hopkins - (\(H\)) √≥ m√©todos visuales usando gr√°ficos de mapa de calor √≥ el biplot de los UE.

El Estad√≠stico Hopkins(Lawson y Jurs 1990): Mide la fiabilidad en base a la probabilidad de que un conjunto de datos dado sea generado por una distribuci√≥n de datos uniforme.En otras palabras, prueba la aleatoriedad espacial de los datos.

\(H_0:\) el conjunto de datos se distribuye uniformemente (es decir, no hay agrupaciones significativas).

\(H_1:\) el conjunto de datos no est√° distribuido uniformemente (es decir, hay agrupaciones significativas).

Para su interpretaci√≥n se toma como umbral un valor de 0.5 no rechazando la hip√≥tesis nula si la estad√≠stica de Hopkins (H) es menor 0.5; en tal caso es poco probable que la matriz tenga grupos estad√≠sticamente significativos; de lo contrario si el valor de la estad√≠stica de Hopkins est√° cerca de 1, entonces podemos rechazar la hip√≥tesis nula y concluir que el conjunto de datos es significativamente agrupable.

Calularemos el estad√≠stico de Hopkins usando la funci√≥n "get_clust_tendency()" indicando primero la matriz de datos, seguido de los grados del libertad (n-1).

Code 
get_clust_tendency(A,n = nrow(A)-1,graph = F)

$plot

Se puede ver que el conjunto de datos es altamente agrupable, (\(H =0.76 > 0.5\)).

Para la evaluaci√≥n visual de la tendencia de agrupamiento se requiere primero calcular la matriz de disimilitud entre observaciones usando la funci√≥n "vegdist()" y luego se gr√°fica con la funci√≥n "fviz_dist()" El nivel de color es proporcional al valor de la diferencia entre las observaciones en rojo tienen alta similitud (es decir, baja disimilitud) y en azul tienen similitud baja. Objetos pertenecientes al mismo grupo se muestran en orden consecutivo.
